:computer: The final project brings together the different phases of web application development, starting with the fundamentals of version control using Git and GitHub, progressing through the creation of static and interactive web pages with HTML, CSS, and JavaScript, and culminating in the development of a more complex application using Angular, TypeScript, and Bootstrap for an internal dashboard system.:bar_chart:.

## üîó How the sprints are connected

The development track was divided into several sprints, each with specific objectives and deliverables that, when integrated, form the foundation of this final project.

*   **Sprint 2 (Desafio_da_Sprint2):**  This sprint focused on the introduction to Git, GitHub, and the SCRUM agile methodology, establishing the foundation for version control. Simple changes were made to HTML files to practice the versioning workflow, consolidating good coding management practices from the beginning.

*   **Sprints 4, 5 e 6 (Desafio_da_Sprint4, Desafio_da_Sprint5, Desafio_da_Sprint6):** üèóÔ∏è These sprints focused on the progressive development of a website for Ford, specifically centered around the Ford Ranger, and later expanded into a concept for the official Ford Brazil website. Responsiveness was carefully considered from the beginning.
    *  **Sprint 4** introduced frontend development with **HTML5, CSS3, and basic JavaScript**, focusing on mobile-first design, interactivity, and semantics. The project was a presentation website for the New Ford Ranger 2022, featuring sections for specifications and a contact form.
    *  **Sprint 5** continued the development, enhancing the Ford Ranger website with more interactivity and, crucially, adding **JavaScript validations** \:heavy\_check\_mark: to the forms, improving the user experience and data integrity.
    *   **Sprint 6** took the project to a new level by recreating a concept for the official Ford Brazil website. It maintained the use of **HTML, CSS, and JavaScript**, but with richer features, such as an interactive image carousel on the homepage, a launches page with vehicle comparisons, and a contact form with validations, including considerations for LGPD (General Data Protection Law).

*   **Sprint 7 (sprint-dash_Ford):** :rocket:  In this sprint, a full-stack application was developed using Angular, TypeScript, and Bootstrap on the frontend, and Node.js with Express.js on the backend. The result was an internal dashboard for Ford employees, featuring user authentication, vehicle metrics display, and VIN lookup, consolidating expertise in modern, data-driven web applications.

:arrow_right: The `Sprint-Final` repository serves as a point of aggregation and documentation of this evolutionary journey, where each sprint contributed with a set of skills and a software artifact that, together, illustrate a comprehensive learning experience in frontend development.

#### :large_blue_circle: In this current sprint, the integration of the two applications was carried out, leading to the complete migration of the website system to Angular.

Considering the set of sprints, various functionalities were implemented, demonstrating a broad range of frontend development skills:

* **Website Structure and Styling:** Creation of responsive and visually appealing layouts for websites üì±, such as the Ford Ranger website and the concept of the official Ford Brazil site. This included headers, footers, content sections, image galleries, and forms.
* **Interactivity with JavaScript:** Implementation of mobile menus, image carousels with automatic and manual navigation, hover effects, and dynamic content manipulation on the page.
